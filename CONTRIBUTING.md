# Contributing

Thanks for taking the time to improve CDLI projects.

## Pull Requests

- Open a pull request from a topic branch.
- Keep changes focused and explain the problem being solved.
- Include tests, examples, or verification notes when behavior changes.
- Wait for CODEOWNER review before merge.

Default branches are protected across CDLI repositories. Direct pushes are restricted, and review is required before changes land.

## Issues

Use public issues for reproducible bugs, documentation gaps, and scoped feature proposals. Do not post secrets, private data, customer data, or vulnerability details in public issues.

## Style

Prefer small, readable changes over broad rewrites. Follow the patterns already present in the repository you are changing.
