# Security Policy

CDLI treats security reports as private operational issues.

## Reporting

Please do not open public issues for vulnerabilities, leaked secrets, exposed credentials, or production security concerns.

Send reports to:

- info@cdli.ai

Include the affected repository or system, a short impact summary, reproduction steps if available, and any logs or screenshots that help us verify the issue.

## Response

We triage security reports by impact and exploitability. If a report affects production systems, credentials, customer data, or private repositories, we prioritize containment before public disclosure.

## Scope

Public repositories in the `cdliai` organization are in scope unless a repository states otherwise. Private products, customer deployments, and internal infrastructure are not available for unsolicited testing.
